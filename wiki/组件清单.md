# 组件清单

本页汇总 `xiaozhiMCP-components` 当前注册在 `index.json` 中的全部组件，便于快速判断组件用途、平台兼容性、配置项和工具列表。

## 总览表

| 组件 | 版本 | 平台 | 依赖 | 工具数量 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `apple_music_macos` | `0.1.0` | MacOs | `git`、`python3`、`pip`、外部仓库 `epheterson/mcp-applemusic` | 动态发现 | macOS Apple Music MCP 自动安装与桥接 |
| `clipboard` | `0.1.0` | Windows | `pywin32` | 2 | Windows 剪贴板读写，支持 CF_HTML 富文本 |
| `exa` | `0.1.0` | Windows / Linux / MacOs | `requests`、Exa API Key | 1 | Exa Web Search API 搜索组件 |
| `local_schedule` | `0.1.0` | Windows / Linux / MacOs | Python 标准库 `sqlite3` | 6 | 本地 SQLite 日程和空闲时间管理 |
| `windows_manager` | `0.1.0` | Windows | `pywin32`、PowerShell | 5 | Windows 窗口枚举、查找、聚焦、关闭和性能信息 |

## `exa`

| 项 | 内容 |
| --- | --- |
| 目录 | `exa/` |
| 入口 | `exa/component.py` |
| 组件类 | `ExaSearchComponent` |
| 工厂函数 | `build_component(config)` |
| 配置段 | `[exa]` |
| 环境变量 | `EXA_API_KEY`、`EXA_BASE_URL` |

工具：

| 工具 | 说明 |
| --- | --- |
| `exa_web_search` | 调用 Exa `/search` 接口，支持搜索类型、内容模式、最大字符数、时效过滤、包含/排除域名、分类、摘要 query、结构化输出 schema 和 system prompt |

最小配置：

```toml
[exa]
api_key = "your_exa_api_key"
base_url = "https://api.exa.ai"
```

示例调用参数：

```json
{
  "keywords": "XiaozhiMCP component architecture",
  "num_results": 5,
  "search_type": "auto",
  "content_mode": "highlights",
  "max_characters": 1400
}
```

| 项 | 内容 |
| --- | --- |
| 目录 | `clipboard/` |
| 入口 | `clipboard/component.py` |
| 组件类 | `ClipboardComponent` |

工具：

| 工具 | 说明 |
| --- | --- |
| `clipboard_set` | 写入 Windows 剪贴板，可同时写入纯文本和 HTML 富文本 |

示例调用参数：

```json

  "content_html": "<b>会议结论</b>：<span style=\"color:green\">同意上线</span>"

```

## `windows_manager`

| 项 | 内容 |
| --- | --- |
| 目录 | `windows_manager/` |
| 入口 | `windows_manager/component.py` |
| 组件类 | `WindowsManagerComponent` |
| 配置段 | `[windows_manager]`，无必填配置 |
| 平台 | Windows |

工具：

| 工具 | 说明 |
| --- | --- |
| `windows_list_open_apps` | 列出可见顶层窗口，结果按标题排序并去重 |
| `windows_find_apps` | 按窗口标题关键词进行大小写不敏感搜索 |
| `windows_focus_app` | 按关键词和匹配序号聚焦窗口，内置三种激活策略 |
| `windows_close_app` | 按关键词和匹配序号发送 `WM_CLOSE` 关闭窗口 |
| `windows_list_app_performance` | 列出窗口对应进程的 CPU 秒数和工作集内存 MB |

示例调用参数：

```json
{
  "keyword": "Chrome",
  "match_index": 1
}
```

## `local_schedule`

| 项 | 内容 |
| --- | --- |
| 目录 | `local_schedule/` |
| 入口 | `local_schedule/component.py` |
| 数据层 | `LocalScheduleStore` |
| 组件类 | `LocalScheduleComponent` |
| 配置段 | `[local_schedule]` |
| 默认数据库 | `~/.xiaozhi_mcp/local_schedule/local_schedule.sqlite3` |

工具：

| 工具 | 说明 |
| --- | --- |
| `schedule_list_events` | 按可选时间范围列出日程事件 |
| `schedule_add_event` | 新增 `range` 或 `deadline` 类型事件 |
| `schedule_update_event` | 更新事件标题、类型和时间字段 |
| `schedule_update_status` | 更新状态，允许值为 `未开始`、`进行中`、`已完成` |
| `schedule_delete_event` | 按 ID 删除事件 |
| `schedule_find_free_slots` | 在指定范围内根据忙碌日程合并结果查找空闲时间段 |

配置示例：

```toml
[local_schedule]
db_path = "~/.xiaozhi_mcp/local_schedule/local_schedule.sqlite3"
```

新增范围事件：

```json
{
  "title": "产品评审",
  "schedule_type": "range",
  "start_time": "2026-05-10T09:00:00",
  "end_time": "2026-05-10T10:00:00",
  "status": "未开始",
  "description": "评审本周发布范围"
}
```

新增截止日期事件：

```json
{
  "title": "提交上线报告",
  "schedule_type": "deadline",
  "due_time": "2026-05-10T18:00:00",
  "status": "进行中"
}
```

## `apple_music_macos`

| 项 | 内容 |
| --- | --- |
| 目录 | `apple_music_macos/` |
| 入口 | `apple_music_macos/component.py` |
| 组件类 | `Component` |
| 配置段 | `[apple_music_macos]` |
| 角色限制 | 只支持 `client`，且 `platform.system().lower() == "darwin"` |

能力：

| 能力 | 说明 |
| --- | --- |
| 自动克隆 | 首次运行克隆 `https://github.com/epheterson/mcp-applemusic.git` |
| 自动更新 | `update_on_startup = true` 时启动发现阶段执行 fetch 和 ff-only pull |
| 自动安装 | 创建虚拟环境并执行 `pip install -e .` |
| 工具代理 | 通过 stdio MCP 协议发现上游工具，并统一添加 `apple_music_` 前缀 |

配置示例：

```toml
[apple_music_macos]
enabled = true
install_dir = "~/.xiaozhi/applemusic-mcp"
update_on_startup = true
```

## 选择建议

| 需求 | 推荐组件 |
| --- | --- |
| 需要联网搜索、摘要或限定域名搜索 | `exa` |
| 需要把内容复制到 Windows 剪贴板 | `clipboard` |
| 需要控制 Windows 桌面窗口 | `windows_manager` |
| 需要本地日程管理和空闲时间查询 | `local_schedule` |
| 需要在 macOS 上操作 Apple Music | `apple_music_macos` |
