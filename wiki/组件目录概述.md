# 组件目录概述

`xiaozhiMCP-components` 是 XiaozhiMCP 的可插拔组件仓库。每个组件都是一个独立目录，通过统一入口文件 `component.py` 暴露 MCP 工具，让主仓库可以在不修改核心系统的情况下扩展搜索、剪贴板、窗口管理、日程和第三方 MCP 桥接能力。

## 适用边界

一个能力适合放在本仓库作为组件，通常满足以下条件之一：

| 条件 | 示例 |
| --- | --- |
| 封装外部 API 或第三方 MCP 服务 | `exa` 封装 Exa Web Search API，`apple_music_macos` 桥接 `mcp-applemusic` |
| 依赖特定操作系统能力 | `clipboard` 依赖 Windows 剪贴板 API，`windows_manager` 依赖 Win32 窗口 API |
| 是可选领域能力并拥有独立数据 | `local_schedule` 使用本地 SQLite 保存日程 |
| 引入核心系统不需要的额外依赖 | `exa` 需要 `requests`，Windows 组件需要 `pywin32` |

如果能力只是核心模块的一层很薄封装，或者属于每个部署都必须具备的基础设施，应该放回主仓库而不是作为组件维护。

## 核心文件职责

| 文件 | 职责 |
| --- | --- |
| `index.json` | 组件目录索引，声明名称、描述、版本、路径、入口、README 和支持平台 |
| `<component>/component.py` | 组件运行入口，实现发现、注册、导出和可选的本地调用逻辑 |
| `<component>/README.md` | 面向使用者的组件说明，必须以 `Platforms: ...` 作为最后一行 |
| `tests/` | 放置与组件行为相关的自动化测试，目前覆盖 `local_schedule` 数据层 |
| `pyproject.toml` | 配置 pytest 搜索路径和 ruff 基础规则 |

## 组件发现约定

运行时读取组件目录后，会按以下顺序在 `component.py` 中寻找入口：

1. `build_component(config: dict, full_config: dict) -> object` 工厂函数。
2. 名为 `Component` 的类。
3. 任何带有 `register` 方法的类。

组件实例通常接收两个配置对象：

```python
config = {
    "enabled": True,
    "api_key": "..."
}

full_config = {
    "server": {"host": "127.0.0.1"},
    "exa": config
}
```

`config` 是组件自己的配置段，`full_config` 是完整配置文件，适合需要跨段读取配置的桥接组件。

## 必需接口

每个组件至少需要实现两个方法：

```python
def register(self, mcp) -> None:
    """使用 @mcp.tool() 将工具注册到 FastMCP 服务端。"""

def export_tools(self) -> list[dict]:
    """返回远程集群注册所需的工具定义。"""
```

`export_tools()` 返回的数据应使用统一结构：

```python
{
    "name": "component_action",
    "description": "工具的人类可读说明",
    "input_schema": {
        "type": "object",
        "properties": {
            "keyword": {"type": "string", "description": "搜索关键词"}
        },
        "required": ["keyword"]
    }
}
```

## 可选接口

```python
def supports_role(self, role: str) -> bool:
    """按角色启用组件，role 通常是 server 或 client。"""

def invoke_tool(self, tool_name: str, arguments: dict) -> dict:
    """同步调用工具，常用于 client 侧或集群远程调用。"""

async def ainvoke_tool(self, tool_name: str, arguments: dict) -> dict:
    """异步调用工具。"""

def set_remote_invoker(self, fn) -> None:
    """接收集群服务端注入的远程调用函数。"""
```

## 当前组件概览

| 组件 | 平台 | 类型 | 核心能力 |
| --- | --- | --- | --- |
| `exa` | Windows / Linux / MacOs | API MCP | 调用 Exa 搜索接口，支持搜索类型、内容模式、域名过滤和结构化输出 |
| `clipboard` | Windows | System MCP | 读写 Windows 剪贴板，支持纯文本和 CF_HTML 富文本 |
| `windows_manager` | Windows | System MCP | 枚举、查找、聚焦、关闭窗口，并读取进程 CPU/内存信息 |
| `local_schedule` | Windows / Linux / MacOs | Data MCP | 基于 SQLite 的本地日程、截止日期、状态和空闲时间查询 |
| `apple_music_macos` | MacOs | MCP Bridge | 自动准备并代理 `epheterson/mcp-applemusic` 工具，统一添加 `apple_music_` 前缀 |

## 通用配置示例

```toml
[exa]
enabled = true
api_key = "your_exa_api_key"
base_url = "https://api.exa.ai"

[local_schedule]
enabled = true
db_path = "~/.xiaozhi_mcp/local_schedule/local_schedule.sqlite3"

[clipboard]
enabled = true

[windows_manager]
enabled = true

[apple_music_macos]
enabled = true
install_dir = "~/.xiaozhi/applemusic-mcp"
update_on_startup = true
```

## 平台标记

组件 README 最后一行必须使用如下格式：

```text
Platforms: Windows|Linux|MacOs
```

支持的平台值只有 `Windows`、`Linux`、`MacOs`。该行会被目录系统解析，用于判断当前平台是否可以安装或启用组件。
